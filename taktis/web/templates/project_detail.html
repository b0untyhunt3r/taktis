Discuss & Research

Optional prep steps you can run on individual tasks before executing them.

D — Discuss

Opens an interactive chat about how to implement the specific task. Identifies gray areas — implementation approach, library choices, data structures — and asks your preference for each. Decisions are saved to DISCUSS_{task_id}.md and injected into the task's context.

R — Research

Runs a non-interactive task that investigates domain-specific concerns for the task: standard patterns, recommended libraries, code examples, common pitfalls, and testing strategies. If you ran Discuss first, your locked decisions are respected. Output saved to RESEARCH_{task_id}.md.

Both files are automatically injected into the task's context when it runs. Neither is required — skip them for straightforward tasks.
